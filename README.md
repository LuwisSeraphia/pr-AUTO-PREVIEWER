# AUTO previewer

PR pre-review toolkit for Python projects. It mines historical PRs to distill review rules, applies those rules to new diffs, and emits concise reports (rules hit, suggested fixes, references). Scripts are token-aware and runnable end-to-end without cloning target repos.

## Usage

### What it does
- Fetches closed PRs from the configured GitHub repo (Python diffs only) and stores them as structured JSON records.
- Generates leveled review rules (`rules.md`) from those records (MUST/SHOULD/MAY, categorized).
- Runs rule checks against local uncommitted changes (raw `git diff`) and renders a report via `report_template.md`.
- Supports incremental rule updates so newly seen patterns keep the rules fresh.

### Environment and Configuration
- Copy and edit `config.json` (or edit directly), focusing on:
  - `github.owner` / `github.repo`: target repository.
  - `github.token_env`: env var name for the GitHub token (default `GITHUB_TOKEN`).
  - `paths.*`: data and output directories.
  - `batch.*`: pagination, rate limits, max record count, etc.
- Export the GitHub token before running:
  ```bash
  export GITHUB_TOKEN=ghp_xxx    # match the token_env value in config.json
  ```
- `config.py` merges defaults and resolves relative paths to absolute—no manual path handling needed.

### Dependencies
- Python 3.8+ with standard library modules used across the scripts: `pathlib`, `json`, `subprocess`, `urllib.request`/`urllib.error`/`urllib.parse`, `datetime`, `functools`, `random`, `time`, `typing`, `pytest` (for tests).
- No third-party runtime dependencies are required for the fetch/update/preview scripts beyond a valid GitHub token; ensure network access to the GitHub API.

### Skills
Full inputs/outputs are in each skill’s `SKILL.md`; quick view below.

- **initialize**  
  Input: `dataset/PR-processed/*.json`. Output: `rules.md`. Purpose: extract actionable MUST/SHOULD/MAY rules from historical PRs.

- **update-rules**  
  Input: staged PR JSONs in `dataset/PR-unprocessed/`. Output: updated `rules.md` and files moved to `dataset/PR-processed/`. Purpose: incrementally evolve the rule set with new patterns. (Runner skips fetch when `PR-unprocessed` already has files.)

- **preview**  
  Input: local `git diff` and current `rules.md`. Output: report via `templates/report_template.md`. Purpose: scan local diffs and list rule hits/violations.

- **skill-creator / skill-installer**  
  Guidance for authoring or installing skills.

### Run Tips (skill commands + bash)
- Initialize rules from scratch (fetch closed PRs, write `dataset/PR-processed/*.json`, generate `rules.md`):  
  ```bash
  /initialize initialize the rules for me according to the requirements
  ```
- Stage new PRs for incremental updates (skip fetch if `dataset/PR-unprocessed/` already has files):  
  ```bash
  PYTHONPATH=. python tools/pre-review/skills/update-rules/runner.py
  ```
- Update rules with staged PRs (consume `dataset/PR-unprocessed/*.json`, merge into `rules.md`, move to `dataset/PR-processed/`):  
  ```bash
  /update-rules update rules for me
  ```
- Preview local changes (scan current `git diff` with `rules.md`, render report via `templates/report_template.md`):  
  ```bash
  /preview review my local changes：<link or local raw git diff path>
  ```
